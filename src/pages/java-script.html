Wstęp do JavaScript'u

JavaScript uruchamia się zaraz po załadowaniu strony. Na pierwszy ogień poznaliśmy z czego składa się:

  • Wartości - dla JS są tym czym liczby dla matematyki. Nie tworzy ich, raczej sie do nich nawiązujemy.
  • Operacje porównania
  • Literały
  • Zmienne
  • Obiekty
  • Funkcje

Pliki zawierające kod JS mają rozszerzenie .js ale podobnie jak style - mają swój znacznik <script></script>;

Poznaliśmy sposób w JS'ie zapisu:

Zmiennej:

const name='Paweł'; - zmienna name: Paweł
const age=34; - zmienna age o wartości 34

Komendę wywołującą zapis w konsoli przeglądarki:

console.log(name) - zwróci w konsoli wartość zmiennej name czyli napis Paweł
console.log(age) - zwróci w konsoli wartość zmiennej age czyli 34

Komendę alert, która wyświeli w osobnym okieny wyświetli zdefiniowany komunikat:
alert('Hejka")- wyświeli komunikat Hejka
Specjalny sposób zapisu literału, czyli tekstu, w kórym umieszczone są zmienne:
`Cześć :) Nazywam się ${name} i mam ${age} lat`; gdzie ${zmienna} to zapis odwołujący się do zmiennej.

Z rzeczy do zapamiętania:

  • Java to nie javaScript
  • javaScript implementuje rzeczy ze specyfikacju ECMA Script
  • javaScript to frontend'owe programowanie

JavaScript: Typy danych i immutability

Zmienne

Zmienne to takie połączenie z naszym wszechświatem wartości. Zmienna to nie wartość, ona tylko wskazuje nam jakąś wartość, która istnieje. Wyróżniamy trzy rodzaje zapisu zmiennych w JS: let, const, var.

  • const – zmienne mające stałe połączenie z jedną wartością,
  • let – zmienne, których połączenie możemy przepisać, ale nazwę tej zmiennej definiujemy tylko raz.

Trzeba pamiętać, że zmienne w JS są jak młody pelikan, „łykną” wszystko. Dlatego trzeba uważać przy ich definiowaniu, żeby czegoś nie nadpisać, lub wywołać jakiś konflikt.

Typy proste w JS (primitives)

Typy proste zależą od wartości przypisanej do danej zmiennej i wyróżniamy:

  • string – tekst, otaczane ‘ lub „ (zalecane ‘),
  • numer – liczba,
  • bigint – duża liczba, większa niż 253,
  • boolean – true lub false,
  • null – wartość celowo niezdefiniowana,
  • undefined – brak definicji,
  • *symbol – tworzy „unikalne opakowanie”.

Polecenie typeof newName – zwraca nam jakiego typu jest zmienna newName.

Jeśli zmiennej y przypiszemy wartość zmiennej x, to po zmianie wartości zmiennej x np. z 10 na 20 to dalej y będzie miało wartość 10.

Immutability (niezmienność)

Nie możemy zmieniać wartości primitive value w JS. To co możemy zmienić, to połączenie między nimi a zmiennymi.

JS: funkcje

Funkcja składa się z serii poleceń zgrupowanych ze sobą i przeznaczonych do wykonywania określonego zadania.

function calculate(myNumber){
return myNumber*7;
}

Wyżej zapisana funkcja zwraca nam rozwiązanie funkcji f(x)=x∙7.Na razie nie zadzieje się nic przy takim zapisie. Nie jest ona nigdzie wywołana. Dopiero jak wpiszemy: calculate(1), to dostaniemy wynik z działania. Żeby wyświetlić wynik z działania funkcji zapisujemy:

const myResult=calculate(1);
console.log(myResult);

Jeśli do funkcji dopiszemy console.log(`Dostałem: ${myNumber}`);, to w konsoli otrzymamy: Dostałem: 1

Funkcje piszemy po to, żeby zautomatyzować czynności (np. gdy daną rzecz trzeba kilka razu wykonać w kodzie).

Funkcje w JS:

  1. Deklarujemy nazwę funkcji (nie zawsze),
  2. W środku dokonujemy obliczeń (wykonujemy zadania),
  3. Funkcja przyjmuje argumenty (ale nie musi),
  4. Funkcja zwraca zawsze coś ( nawet jeśli nie mamy return – wtedy funkcja zwraca undefined).

Funkcje: Kiedy i dlaczego stosujemy?

  • Deklarujemy kod, który się powtarza (zasada DRY – don’t repeat yourself),
  • Zwiększamy czytelność kodu,
  • Łatwiejsze szukanie błędów,
  • Odpowiedzialność za wykonanie zadania w jednym miejscu

Istnie również skrócony zapis funkcji w JS, tzw. (Fat) arrow functions (=>)

const calculate = (myNumber) => {
myNumber=mynumber+3;,
console.log(myNumber);
return myNumber*7;
}

Wtedy wyświeli się nam rozwiązanie funkcji: calculate(2); //35

Możliwy jest też jeszcze krótszy zapis:

const calculate = (myNumber) => myNumber*7;

(Fat) arrow function:

  • skraca zapis deklaracji funkcji,
  • zwiększa czytelność (dyskusyjne),
  • nie tworzy nowego kontekstu this, nie ma dostępu do arguments i super,
  • nie jest zamiennikiem „tradycyjnej” funkcji (ale w większości przypadków nie zobaczymy różnicy).

JS: obiekt

Obiekt jest strukturą danych. W obiekcie zmienne nazywamy własnościami (property), a funkcje to metody (methods).
Przykładowy obiekt:

const deathStar = {
diameter: 120000,
fire: (target) => {
console.log(`${target} destroyed 💥`)
},
isOperating: true,
levels: 357,
name: 'Death Star',
population: 10000,
} const obiekt = {
klucz: wartość,
}
klucz
jest unikalny w ramach obiektu,
wartość
to prymitywy, funkcje, zagnieżdżenia.

Żeby wywołać w kodzie interesujący nas klucz z obiektu stosujemy „.” (dot notation), czyli zapisujemy to odwołanie jako: nazwaObiektu.klucz;.
Innym sposobem do odwołania się do własności obiektu (ale nie jego metod) jest bracket notation []. Czyli nazwaObiektu[‘klucz’];.

JS: Immutablility w obiektach (niemutowalność)

Dla przypomnienia: nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenie między nimi, a zmiennymi.
Połączenie do obiektu dla const jest sztywne, ale w obiekcie, dalsze połączenia do wartości są zmienne. Czyli z tego wynika, że obiekty złożone mogą mieć przypisane różne własności do wartości, a obiekty proste dla const są stałe.
Obiekty tworzymy w przeciwieństwie do wartości, do których się podłączamy.
W obiektach, gdy odwołujemy się do obiektu zagnieżdżonego w innym obiekcie, to przy zmianie właściwości obiektu zagnieżdżonego nie zmieniamy połączenia do tego obiektu. Skutkuje to tym, że i w tym obiekcie będą wyświetlane zmienione odwołania do wartości.
