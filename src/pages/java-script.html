Wstęp do JavaScript'u

JavaScript uruchamia się zaraz po załadowaniu strony. Na pierwszy ogień poznaliśmy z czego składa się:

  • Wartości - dla JS są tym czym liczby dla matematyki. Nie tworzy ich, raczej sie do nich nawiązujemy.
  • Operacje porównania
  • Literały
  • Zmienne
  • Obiekty
  • Funkcje

Pliki zawierające kod JS mają rozszerzenie .js ale podobnie jak style - mają swój znacznik <script></script>;

Poznaliśmy sposób w JS'ie zapisu:

Zmiennej:

const name='Paweł'; - zmienna name: Paweł
const age=34; - zmienna age o wartości 34

Komendę wywołującą zapis w konsoli przeglądarki:

console.log(name) - zwróci w konsoli wartość zmiennej name czyli napis Paweł
console.log(age) - zwróci w konsoli wartość zmiennej age czyli 34

Komendę alert, która wyświeli w osobnym okieny wyświetli zdefiniowany komunikat:
alert('Hejka")- wyświeli komunikat Hejka
Specjalny sposób zapisu literału, czyli tekstu, w kórym umieszczone są zmienne:
`Cześć :) Nazywam się ${name} i mam ${age} lat`; gdzie ${zmienna} to zapis odwołujący się do zmiennej.

Z rzeczy do zapamiętania:

  • Java to nie javaScript
  • javaScript implementuje rzeczy ze specyfikacju ECMA Script
  • javaScript to frontend'owe programowanie

JavaScript: Typy danych i immutability

Zmienne

Zmienne to takie połączenie z naszym wszechświatem wartości. Zmienna to nie wartość, ona tylko wskazuje nam jakąś wartość, która istnieje. Wyróżniamy trzy rodzaje zapisu zmiennych w JS: let, const, var.

  • const – zmienne mające stałe połączenie z jedną wartością,
  • let – zmienne, których połączenie możemy przepisać, ale nazwę tej zmiennej definiujemy tylko raz.

Trzeba pamiętać, że zmienne w JS są jak młody pelikan, „łykną” wszystko. Dlatego trzeba uważać przy ich definiowaniu, żeby czegoś nie nadpisać, lub wywołać jakiś konflikt.

Typy proste w JS (primitives)

Typy proste zależą od wartości przypisanej do danej zmiennej i wyróżniamy:

  • string – tekst, otaczane ‘ lub „ (zalecane ‘),
  • numer – liczba,
  • bigint – duża liczba, większa niż 253,
  • boolean – true lub false,
  • null – wartość celowo niezdefiniowana,
  • undefined – brak definicji,
  • *symbol – tworzy „unikalne opakowanie”.

Polecenie typeof newName – zwraca nam jakiego typu jest zmienna newName.

Jeśli zmiennej y przypiszemy wartość zmiennej x, to po zmianie wartości zmiennej x np. z 10 na 20 to dalej y będzie miało wartość 10.

Immutability (niezmienność)

Nie możemy zmieniać wartości primitive value w JS. To co możemy zmienić, to połączenie między nimi a zmiennymi.
