CSS czyli kaskadowy arkusz stylów (ang. Cascading Style Sheets)

Tydzień II 20.04.-26.04.2020 r.

W tym tygodniu poznaliśmy, że CSS odpowiada za prezentacje dokumentu. Tag <style>, który odpowiada za CSS umieszczamy w sekcji <head>. Innym sposobem jest przeniesienie CSS do innego pliku. Ma to na celu zwiększenie przejrzystości kodu w dokumencie. Możemy to zapisać za pomocą komendy:

<link href="style.css" rel="stylesheet" type="text/css">
<link>
komenda przesyła dokument po jakieś źródło do innego pliku np. jak w naszym przypadku po arkusz stylów CSS. Również zamieszczamy tą komendę w sekcji <head>

Dane własności, które chcemy zmienić w dokumencie poprzez CSS, zapisujemy:

selector{
właściwość: wartość;
}

Poznane właściwości w tym tygodniu to:

color:
zmienia kolor tekstu
background:
zmienia kolor tła

Tydzień III 27.04.-02.05.2020 r.

Wygląd dokumentu możemy zmieniać również poprzez atrybut style umieszczony bezpośrednio w tagu. Nazywamy go stylem inline (liniowy). Zapisujemy go w następujący sposób:

<znacznik style="właściwość: wartość;">treść</znacznik>

Zmienianie własności CSS możliwe jest poprzez:

klasy (.class)

<znacznik class="name">treść</znacznik>

.name{
właściwość: wartość;
}

Takich samych klas możemy używać dla różnych znaczników i każdy z nich będzie miał takie same właściwości.

identyfikator (#id)

<znacznik id="name">treść</znacznik>

#name{
właściwość: wartość;
}

Identyfikator możemy użyć dla jednego konkretnego znacznika. Nie powinien odnosić się do innych znaczników.

Jeśli chcemy zmienić właściwości danego elementu strony bez nadawania mu klasy lub identyfikatora (np. chcemy żeby odnośniki w <nav> były innego koloru niż na reszcie strony) to możemy łączyć selektory. Możemy to zapisać:

nav a{
właściwość: wartość;
}

Dzięki temu będzie możliwe zmienienie koloru odnośników <a>; tylko w nawigacji <nav>;

Kiedy do wartości dla danego selektora dopiszemy po spacji !important, spowodujemy, że będzie ona stosowana jako pierwsza pod uwagę.

Specyficzność w CSS definiuje co będzie pierwsze użyte (ważność) jeśli do danego elementu może się odwoływać kilka selektorów. Przedstawia się ona w następujący sposób:

  1. !important
  2. style=""
  3. #id
  4. .name
  5. element

Żeby uniknąć problemów, z specyficznością stylów najlepiej stosować klasy. Unikniemy wtedy potrzebę przeliczania, żeby dowiedzieć się jaki styl otrzyma dany element
