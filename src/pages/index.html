Jak stworzyć prawidłowego HTLM'a?

To jest właśnie pytanie, na które zaczynam poznawać odpowiedzi. Drugi tydzień również jest rozgrzewkowy, ale pojawiają się już ambitniejsze treści. Poznajemy już pewne zasady funckjonowania strony internetowej oraz jak kolejne warstwy (czyli tekst, HTML, CSS i JavaScript) wpływają na siebie. Poznajemy podstawową strukturę HTLM, a także jak można edytować podstawowe elementy wyglądu strony poprzez CSS. Dodatkowo odkrywamy, co łączy programowanie z magią. Pozajemy również podstawowe zastasowanie DevTools. Jako jedno z zadań jest rozwijana ta strona, którą właśnie czytasz.

Wartwy strony internetowej

Zaczynamy od poznania poszczególnych warstw dokumentu. W jego skład wchodzi:

  • Warstwa 0: Tekst.
  • Warstwa 1: HTML.
  • Warstwa 2: CSS.
  • Warstwa 3: JavaScript

Poszczególne warstwy mają wpływ na siebie. Na tekst (czyli wszystko co się wyświetla w dokumencie, czyli dodatkowo obrazy, linki, tabele itd.) mają wpływ inne warstwy. HTML nadaje strukturę stronie. Stanowi jej szkielet. CSS odpowiada za wygląd. A w JavaScript dodaje interaktywność do strony. HTML wpływa bezpośrednio tylko na tekst, za to CSS już może wywierać wpływ na kod HTML i tekst. JavaScript może wywołać najwięcej zamieszania. Wpływać już może na: tekst, kod HTML i CSS.

HTML II

Do podstaw, które poznaliśmy w poprzednim tygodniu dochodzą nowe znaczniki. W tym sposób na podstawową organizację nawigacji na stronie.

Nadawanie nazw elementom

Jest to jedna z najtrudniejszych czynności do wykonania. Żeby wprowadzić porządek najlepiej stosować kilka prostych zasad:

  • nazwy bez polskich znaków,
  • nie używamy spacji
  • najlepiej używać angielskich słów opisujących dany element,
  • trzymamy się jednego już istniejącego w dokumencie schematu.

Dla wprowadzenia czytelności danej kilku wyrazowej nazwy zapisanej bez spacji, można użyć jedne kilku istniejących konwencji zapisu:

camelCase
pierwsze słowo z małej litery, a kolejne wyrazy bez spacji zapisane są z dużej,
kebab-case
wszystkie wyrazy zapisujemy z małej litery, a oddzielamy je od siebie za pomocą myślnika "-",
snake_case
wszystkie słowa zapisujemy z małej litery i oddzielamy je "_",
PascalCase
wszystkie słowa zapisujemy z dużej litery, bez spacji między sobą.

Jest Git, a nawet gitHub :)

Git to program do zarządzania rozwijanym projektem oraz system kontroli wersji. Jest to narzędzie,dzięki któremu lepiej pracuje się w grupie i łatwo śledzi się zmiany, ewentualnie je wracać do wcześniejszej wersji. Umożliwia również rozdzielenie wersji i rozwijanie dwóch różnych wariantów projektu, by na koniec móc wybrać lepszą.

Program Git możemy uruchomić w terminalu poprzez polecenie: git. Musimy tylko pamiętać, żeby uruchomić Git'a w katalogu z rozwijanym projektem.

Przed pierwszym użyciem Git'a, trzeba go skonfigurować:

Życie w pudełku

Display

Strony internetowe są tworzone dla dwu wymiarowych wyświetlaczy. Co się z tym wiąże, to sposób projektowania strony. Na początek poznajemy własność:

display: block
elementy ustawiają się jeden pod drugim,
display: inline
elementy ustaiwają się obok siebie. Nie mogą mieć zdefiniowanej wysokości i szerokości,
display: inline-block
własność specjalna, łączy w sobie dwie rzeczy. Zachowuje się jak element liniowy, ale możemy nadawać im własności niedostępne elementom liniowym, a są dostępne dla elementów blokowych.
widht / height
szerokość / wysokość.
max-width / max-height
nie ma sztywnej wysokości / szerokości, tylko wymiar maksymalny.
min-width / min-height
element nie będzie mniejszy niż zadana wartość.
overflow: hidden
pozwala na odcięcie tej treści, która występuje poza wielkość kontenera. Odcięta treść nie będzie widoczna.
overflow: visible
domyślna wartość, powoduje, że odcięta treść wystaje poza kontener i jest widoczna.
overflow: auto
dodaje nam scroll dzięki, któremu możemy przewijać zawartość kontenera. Po przewinięciu mamy dostęp do całego tekstu, a on nie wychodzi poza kontener.

Jednostki

Jednostki stosowane w własnościach CSS’owych to przede wszystkim:

  • piksele (px) np. 600px,
  • procenty (%) np. 60%,

Ponieważ w forntendzie definiując szerokość i wysokość elementu nie jesteśmy wstanie przewidzieć ilości treści i boxy muszą się przesuwać to wprowadzono następujące jednostki:

viewport width (vw)
określa szerokość elementu w procentach, względem całego okna np. 60vw,
viewport height (vh)
analogicznie jak dla wysokości.

Box Model – fundament budowy layout’u strony inernetowej.

padding
wewnętrzny margines – określa o ile zawartość ma być odsunięta od wewnętrznej granicy pudełka,
border
określa ramkę wokół elementu:
border-width:
szerokość ramki, o border-style: - rodzaj ramki (solid – ciągła, dashed – wykreskowana, dotted – wykropkowana),
border-radius: 10px;
własność pozwala na zaokrąglenie rami (obramowania) 10px określa promień zaokrąglenia. Jeśli wpiszemy 50%, to zrobimy koło z naszego elementu. Trzeba będzie tylko pamiętać, że treść będzie przycięta w takim elemencie i żeby tego uniknąć, możemy zastosować odpowiedniej wielkości padding.
margin
margines określa, o ile odsuwamy element od innych elementów i działa tylko przy elementach blokowych.

Box-sizing

box-sizing: content-box;
domyślna wielkość. Mowi ona nam, że jeśli zdefiniujemy sobie szerokość, to nie wpływa na nią padding, border, czyli jest tylko szerokością elementu. Z tego wynika, że cały box jest dużo szerszy,
box-sizing: border-box
przy takim zapisie szerokość i wysokość odnoszą się do całego boxa ( uwzględniając margines wewnętrzny oraz ramkę). Dzięki zastosowaniu tej własności, nie trzeba liczyć jaką wielkość ma mieć nasz element, żeby całość miała konkretny wymiar z uwzględnieniem ramki (czyli wszystkiego co należy do elementu).

Position

Positon – pozwala na pozycjonowanie elementów w naszej przestrzeni nie tylko za pomocą narzuconych nam przez przeglądarkę układom. Dzięki tej funkcji możemy powiedzieć przeglądarce, że w tym i tym pikselu ma być ten element. Domyślną wartością dla position jest static czyli

position: relative;
element zależny od pozycji absolute,
position: absolute;
określa dokładne położenie elementu
position: fixed;
ma podobne właściwości jak absolute, tylko jest zawsze wyświetlany na tym samym miejscu, nawet jeśli przesuwamy stronę. Często stosowany jest dla strony,
position: sticky
element na oczątku jest na swoim miejscu, ale gdy scrollujemy (przewijamy) tekst, to element pozostaje w tym samym miejscu, aż do końca danej sekcji. Możemy go użyć jeśli chcemy żeby nagłówek był widoczny przez czas czytania artykułu (sekcji).Na początku nie zachodzi na tekst, dopiero później, przy przesuwaniu tekstu zajmuje stałe miejsce i przykrywa tekst przewijany, aż do końca sekcji.

Float

Float służy do opływania elementów. Nie jest ograniczony, tzn. jest zakotwiczony w treści.

JavaScript: Typy danych i immutability

Zmienne

Zmienne to takie połączenie z naszym wszechświatem wartości. Zmienna to nie wartość, ona tylko wskazuje nam jakąś wartość, która istnieje. Wyróżniamy trzy rodzaje zapisu zmiennych w JS: let, const, var.

const
zmienne mające stałe połączenie z jedną wartością,
let
zmienne, których połączenie możemy przepisać, ale nazwę tej zmiennej definiujemy tylko raz.

Trzeba pamiętać, że zmienne w JS są jak młody pelikan, „łykną” wszystko. Dlatego trzeba uważać przy ich definiowaniu, żeby czegoś nie nadpisać, lub wywołać jakiś konflikt.

Typy proste w JS (primitives)

Typy proste zależą od wartości przypisanej do danej zmiennej i wyróżniamy:

  • string – tekst, otaczane ‘ lub „ (zalecane ‘),
  • numer – liczba,
  • bigint – duża liczba, większa niż 253
  • boolean – true lub false,
  • null – wartość celowo niezdefiniowana,
  • undefined – brak definicji,

Polecenie typeof newName – zwraca nam jakiego typu jest zmienna. Jeśli zmiennej y przypiszemy wartość zmiennej x, to po zmianie wartości zmiennej x np. z 10 na 20 to dalej y będzie miało wartość 10.

Immutability (niezmienność)

Nie możemy zmieniać wartości primitive value w JS. To co możemy zmienić, to połączenie między nimi a zmiennymi.

BEM Block – Element – Modifier

Dlaczego stworzono BEM?

  • Świat stylowania doszedł do wysokiego stopnia skomplikowania (specyficzność).
  • Problem z logicznym nazwaniem klas.
  • Brak ujednoliconego sposobu stylowania.
  • Importanty, nadpisywanie rzeczy.
  • Brak odwzorowania struktury HTML w CSS.
  • Stylowanie kilkoma klasami, powtarzające się nazwy.

Dlatego:

  • Szukamy elementów na stronie i dzielimy je na bloki (np. nawigacje, artykuły).
  • Myślimy o elementach składowych (robimy analizę, selekcję).
  • Tworzymy dwie klasy jeśli element ma modyfikator (w modyfikatorze zmieniamy tylko unikalne dla niego wartości, nie przepisujemy rzeczy z klasy bazowej).

DOM’owe przedszkole DOM czyli Document Object Model (Obiektowy model dokumentu).

  • Dom to złożony w przeglądarce HTML, już po wszystkich zmianach od JS.
  • JavaScript może modyfikować DOM.
  • Funkcja querySelector zwraca węzeł (node) z HTML’a, selktor zapisywany zgodnie z CSS.
  • Korzystając z BEM’a tworzymy dodatkową klasę z modyfikatorem –js (ułatwia identyfikację i usuwa potencjalne problem).
